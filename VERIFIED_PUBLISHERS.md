# Verified Publisher Program

Verified Publishers receive a ✅ badge on all their plugins and priority
placement in marketplace search results.

## Criteria

To be eligible for Verified Publisher status, an author **MUST**:

| # | Requirement | How to verify |
|---|-------------|---------------|
| 1 | **GitHub account is at least 1 year old** | Account creation date |
| 2 | **Two-factor authentication (2FA) enabled** | GitHub account settings |
| 3 | **At least 1 published plugin with trust score ≥ 85** | Automated score check |
| 4 | **No open security incidents on any plugin** | Incident tracker |
| 5 | **Signed DCYFR Verified Publisher Agreement** | Agreement on file |
| 6 | **Plugin has ≥ 10 GitHub stars or 50 downloads** | Automated metrics |

## Application Process

1. Ensure all criteria above are met.
2. Open a GitHub issue with title: `[Verified Publisher] <your-github-handle>`
3. DCYFR will review your plugins and respond within **5 business days**.
4. If approved, you'll receive a co-publisher invite and the ✅ badge.

## Revocation

Verified status may be revoked if:

- A critical security vulnerability is found in a published plugin and not patched within SLA
- Publisher agreement is violated
- 2FA is disabled
